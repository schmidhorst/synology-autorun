Status- und Copy-LED-Anzeige

Autorun kann die Status oder die Copy-LED steuern. Damit kann eine laufende Aktivität und gegebenenfalls ein Fehler angezeigt werden.

Hinweise:

  • Es gibt keine einfache Möglichkeit, den aktuellen LED-Zustand auszulesen oder zurückzukehren zum vorigen Zustand. Wenn die LED-Steuerung in den Einstellungen von Autorun aktiviert ist, dann wird der aktuelle LED-Zustand (z.B. Status orange blinkend wegen einem Fehler) geg. von Autorun überschrieben. Und auch umgekehrt überschreiben andere Ereignisse unter Umständen die von Autorun gesetzten Anzeige.
  • Es gibt Autorun-Konfigurations-Einstellungen, bei der ein Fehler (bzw. wählbare Return-Codes des gestarteten Skripts) nach dem Ende mit einer LED angezeigt wird. Das Löschen der Fehler-Anzeige ist dann nur mit einem als root ausgeführten Skriptkommando (z.B. über die Aufgabenplanung) möglich.

Das Skriptkommand für die LED-Steuerung lautet z.B. echo "7" > /dev/ttyS1

Statt der 7 im Beispiel (für Status LED aus) sind folgende Zeichen zu verwenden:
Code Kommando Aktion
7 echo "7" > /dev/ttyS1 Status LED aus
8 echo "8" > /dev/ttyS1 Status LED grün
9 echo "9" > /dev/ttyS1 Status LED grün blinkend
: echo ":" > /dev/ttyS1 Status LED orange
; echo ";" > /dev/ttyS1 Status LED orange blinkend
@ echo "@" > /dev/ttyS1 Copy LED an
A echo "A" > /dev/ttyS1 Copy LED blinkend
B echo "B" > /dev/ttyS1 Copy LED aus
