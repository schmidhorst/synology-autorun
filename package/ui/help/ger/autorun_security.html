Sicherheit

Speicherort des Skripts:

Es gibt zwei grundsätzlich verschiedene Orte, an denen das vom Autorun-Paket gestartete Skript abgelegt werden kann:

  • Auf den externen Speichergeräten:
    • Nur die Speichergeräte, die mit dem Skript auf diese Weise vorbereitet wurden, starten die Aktion.
    • Alle anderen Speichermedien, die eingesteckt werden können, werden ignoriert.
    • Wenn die vorhandene Sicherheitsoption nicht genutzt wird, kann jeder, der Zugriff auf die Diskstation hat, durch Einstecken eines vorbereiteten USB-Sticks beliebige Dinge tun, z.B. die gesamte Diskstation löschen und freien Zugriff auf das gesamte Netzwerk haben.
  • Intern im DSM:
    • Jedes Mal, wenn ein Speichermedium eingesteckt wird, wird das Skript gestartet. Das Skript selbst muss unter Umständen prüfen, ob es seine Aktionen auf dem aktuellen Speichergerät durchführen soll.
    • Die verfügbare Sicherheitsoption wird in den meisten Fällen nicht erforderlich sein.

Zusätzlich zu dem oben erwähnten primären Skript ist optional auch ein sekundäres Skript möglich, das nach dem Auswerfen des Speichermediums ausgeführt wird. Dieses zweite Skript kann sich natürlich nur intern auf dem DSM befinden. Der Zugriff auf den ausgeworfenen Speicher ist zu dem Zeitpunkt, zu dem es gestartet werden soll, nicht mehr möglich.

Alle Skripte laufen mit Root-Rechten. Intern platzierte Skripte können sich daher fast überall im Dateisystem der DSM befinden. Benutzer ohne Administratorrechte sollten keinen Zugriff haben, zumindest keinen Schreibzugriff!

Sicherheitsoption:

Die Ausführung von Skripten kann auf zuvor registrierte Skripte anhand ihrer Prüfsumme (SHA256-Hash) beschränkt werden. Andere Skripte werden dann nicht ausgeführt.
Es gibt zwei verschiedene Ansätze:

  1. In den Einstellungen bei der Installation des Autorun-Pakets wird die Einstellung "1 erstes Skript erlaubt" gewählt, und die Prüfsumme des ersten Skripts nach der Installation wird ermittelt und registriert. Nur dieses Skript und unveränderte Kopien davon werden ausgeführt. Die Sicherheit wappnet sich automatisch mit dem ersten Skript nach der Erst-, und ggf. auch nach einer Neuinstallation. Sollte das Skript jemals eine Änderung benötigen, so führen Sie anschließend erneut eine Autorun-Paket-Neuinstallation mit dieser Einstellung "1 erstes Skript erlaubt" durch. Die alte Registrierung wird gelöscht und die neue Skriptprüfsumme wird anschließend als die einzig zulässige registriert.
  2. Bei der Paketinstallation wird zunächst "2 Add any hash" ausgewählt. Nun müssen alle externen Speicher, die unterschiedliche Skripte haben, nacheinander mal eingesteckt werden. Damit die benötigten Prüfsummen registriert und hinzugefügt werden können. Danach muss die Installation erneut ausgeführt werden und der Wert auf "3 Only previously registered" geändert werden. Von da an ist die Sicherheit aktiviert.
Verfahren B kann auch verwendet werden, wenn zuvor A verwendet wurde und eine zusätzliche Prüfsumme, z.B. für eine neue Skriptvariante, parallel zu der bereits registrierten Skriptvariante hinzugefügt werden muss.
