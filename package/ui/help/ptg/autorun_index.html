Autorun

O pacote "Autorun" pode desencadear uma acção quando um disco rígido externo é ligado a uma Estação Synology. Isto é muitas vezes muito útil para a cópia automática de dados. Pelo menos na versão 7.1 do DSM, tal opção de evento está em falta no programador de tarefas. Existem apenas "Startup" e "Shutdown" como eventos nas "Tarefas Disparadas", mas nenhum "Armazenamento externo USB inserido" ou semelhante.

Por "Acção" entende-se principalmente a execução de um guião de concha BASH. O script pode então copiar ficheiros entre a estação de disco e o disco usando comandos como o rsync. Ou o script pode, por exemplo, também iniciar uma tarefa definida no agendador de tarefas.

Como acção adicional, o disco também pode ser ejectado após a acção estar concluída. Uma vez que isto significa que os dados no disco externo já não são directamente acessíveis, isto proporciona alguma protecção dos dados no disco externo contra a sua destruição, por exemplo, através de resgates. No entanto, uma vez que o disco também pode ser reintegrado no sistema através de software, esta não é uma protecção completa.

Uma protecção ainda melhor contra os resgates pode ser conseguida da seguinte forma: Utilizar um suporte de dados externo com alimentação própria, como é comum nos discos rígidos USB de 3,5 polegadas. A fonte de alimentação é operada através de uma tomada inteligente. Esta tomada inteligente é depois desligada depois de o disco ser ejectado. Desde que o software de resgate não substitua esta implementação, o disco externo que é desligado do sistema desta forma é protegido. Advertência: Enquanto o disco externo é montado para uma actualização, não existe virtualmente nenhuma protecção contra o ransomware que possa ter-se infiltrado previamente no seu sistema! Por conseguinte, utilize sempre pelo menos dois discos em rotação para backups!
