Seguridad

Ubicación del script:

Existen dos ubicaciones fundamentalmente diferentes donde se puede almacenar el script iniciado por el paquete de ejecución automática:

  • En los dispositivos de almacenamiento externos:
    • sólo los dispositivos de almacenamiento preparados con el script de esta forma iniciarán la acción .
    • Todos los demás dispositivos de almacenamiento que se puedan conectar se ignoran.
    • Si no se utiliza la opción de seguridad disponible, cualquiera que tenga acceso a la estación de disco puede hacer cosas arbitrarias insertando una memoria USB preparada, por ejemplo, borrar toda la estación de disco y tener libre acceso a toda la red.
  • Internamente en DSM:
    • Cada vez que se inserta un medio de almacenamiento, se inicia el script .
    • El propio script puede necesitar comprobar si debe realizar sus acciones en el dispositivo de almacenamiento actual.
    • La opción de seguridad disponible no será necesaria en la mayoría de los casos.

Además del script primario mencionado anteriormente, también es posible tener un script secundario opcional que se ejecute después de que el dispositivo de almacenamiento haya sido expulsado. Por supuesto, este segundo script sólo puede estar ubicado internamente en el DSM. El acceso a la memoria expulsada ya no es posible en el momento en que debe iniciarse.

Todos los scripts se ejecutan con privilegios de root. Por lo tanto, los scripts ubicados internamente pueden localizarse en casi cualquier lugar del sistema de archivos del DSM. Los usuarios sin derechos de administrador no deberían tener acceso, ¡al menos no acceso de escritura!

Opción de seguridad:

La ejecución de scripts puede restringirse a scripts previamente registrados en función de su suma de comprobación (hash SHA256). Los demás scripts no se ejecutarán.
Existen dos enfoques diferentes:

  1. En los ajustes al instalar el paquete de ejecución automática, se selecciona el ajuste "1 primer script permitido", y se determina y registra la suma de comprobación del primer script tras la instalación. Sólo se ejecuta este script y las copias no modificadas del mismo. Security se arma automáticamente con el primer script después de la instalación inicial y, si es necesario, de la reinstalación. Si alguna vez es necesario modificar el script, realice otra reinstalación del paquete de ejecución automática con esta configuración de "1 primer script permitido". El registro antiguo se borra y el nuevo script checksum se registra posteriormente como el único permitido.
  2. Durante la instalación del paquete, se selecciona primero "2 Añadir cualquier hash". Ahora todas las memorias externas, que tienen diferentes scripts, deben ser conectadas una tras otra. Así se pueden registrar y añadir las sumas de comprobación necesarias. Después hay que volver a ejecutar la instalación y cambiar el valor a "3 Sólo registradas previamente". A partir de ese momento, la seguridad está activada.
El procedimiento B también puede utilizarse si antes se utilizó A y debe añadirse una suma de comprobación adicional, por ejemplo, para una nueva variante de script, en paralelo a la variante de script ya registrada.
