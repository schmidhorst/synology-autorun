Autorun

Das "Autorun"-Paket kann eine Aktion auslösen, wenn eine externe Festplatte an eine Synology Station angeschlossen wird. Dies ist oft sehr nützlich für das automatische Kopieren von Daten. Zumindest in DSM Version 7.1 fehlt eine solche Ereignisoption im Taskplaner. Es gibt nur "Startup" und "Shutdown" als Ereignisse in den "Triggered Tasks", aber kein "External USB storage inserted" oder ähnliches.

Mit "Aktion" ist in erster Linie die Ausführung eines BASH-Shell-Skripts gemeint. Das Skript kann dann mit Befehlen wie rsync Dateien zwischen der Diskstation und der Platte hin und her kopieren. Oder das Skript kann z.B. auch einen im Aufgabenplaner definierten Task starten.

Als zusätzliche Aktion kann die Platte nach Abschluss der Aktion auch ausgeworfen werden. Da dadurch die Daten auf dem externen Datenträger nicht mehr direkt zugänglich sind, bietet dies einen gewissen Schutz der Daten auf dem externen Datenträger vor Zerstörung, z.B. durch Ransomware. Da der Datenträger aber auch per Software wieder in das System integriert werden kann, ist dies kein vollständiger Schutz.

Ein noch besserer Schutz vor Ransomware kann auf folgende Weise erreicht werden: Verwenden Sie einen externen Datenträger mit eigener Stromversorgung, wie es bei 3,5-Zoll-USB-Festplatten üblich ist. Die Stromversorgung wird über eine intelligente Steckdose betrieben. Diese Smart-Socket wird dann nach dem Auswerfen des Datenträgers abgeschaltet. Solange Ransomware diese Implementierung nicht außer Kraft setzt, ist der externe Datenträger, der auf diese Weise vom System getrennt wird, geschützt. Warnung: Während der externe Datenträger für ein Update eingebunden ist, besteht praktisch kein Schutz vor Ransomware, die zuvor in Ihr System eingedrungen sein könnte! Verwenden Sie daher immer mindestens zwei Datenträger im Wechsel für Backups!
