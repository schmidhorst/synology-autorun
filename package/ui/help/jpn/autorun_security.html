セキュリティ

スクリプトの場所です：

オートランパッケージで起動するスクリプトの置き場所は、根本的に2つあります：

  • 外部記憶装置上：
    • 外部記憶装置：この方法でスクリプトが準備された記憶装置だけが動作を開始します。
    • もし利用可能なセキュリティオプションが使用されていない場合、ディスクステーションにアクセスできる人なら誰でも、用意されたUSBスティックを差し込むことで任意のことができる。 例えば、ディスクステーション全体を削除してネットワーク全体に自由にアクセスできる。
  • DSMの内部：
    • ストレージデバイスが差し込まれるたびに、スクリプトが起動
される。
    • スクリプトは、現在のストレージデバイスに対してアクションを実行するかどうかをチェックする必要が
あります
      。
    • 利用可能なセキュリティオプションは、ほとんどの場合、必要
ありません。

上記のプライマリー・スクリプトに加えて、ストレージ・デバイスが排出された後に実行されるセカンダリー・スクリプトもオプションで可能である。 このセカンドスクリプトは当然ながらDSMの内部のみに配置することができます。排出されたメモリへのアクセスは、起動する時点ではもう不可能である。

すべてのスクリプトはroot権限で実行されます。したがって、内部的に配置されたスクリプトは、DSMファイルシステムのほぼすべての場所に配置することができます。 管理者権限のないユーザは、少なくとも書き込み権限を持たないようにする必要があります！

セキュリティオプションです：

チェックサム（SHA256ハッシュ）を用いて、あらかじめ登録されたスクリプトに限定して実行することができます。それ以外のスクリプトは実行されません。
2種類のアプローチがあります：

  1. 自動実行パッケージのインストール時の設定で、「1つの最初のスクリプトを許可する」設定を選択し、インストール後の最初のスクリプトのチェックサムを決定して登録します。 このスクリプトとその変更されていないコピーのみが実行されます。初回インストール後、また必要に応じて再インストール後も、最初のスクリプトでセキュリティが自動的に武装されます。 スクリプトを変更する必要がある場合は、この設定「1 first script allowed」で再度オートランパッケージを再インストールしてください。古い登録は削除され、新しいスクリプトのチェックサムが唯一許可されたものとして登録されます。
  2. パッケージのインストール時に、最初に「2 Add any hash」が選択されます。これで、異なるスクリプトを持つすべての外部ストレージを次々に接続する必要があります。そうすれば、必要なチェックサムを登録し、追加することができます。その後、インストールを再度実行し、値を「3 Only previously registered」に変更する必要があります。それ以降、セキュリティは強化されます。
手順 B は、手順 A が以前に使用されており、新しいスクリプトバリアントなどのチェックサムを、以前に登録したスクリプトバリアントと並行して追加する必要がある場合に使用することもできます。
