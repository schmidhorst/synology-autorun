Светодиодный индикатор состояния и копирования

Автозапуск может управлять светодиодным индикатором состояния или копирования. Он может использоваться для индикации выполняемой работы и, при необходимости, ошибки.

Примечания:

  • Не существует простого способа прочитать текущее состояние светодиода или вернуться к предыдущему состоянию. Если управление светодиодами включено в настройках Autorun, то текущее состояние светодиодов (например, статус мигает оранжевым из-за ошибки) может быть перезаписано Autorun. И наоборот, другие события могут перезаписать индикацию, установленную Autorun.
  • Существуют настройки конфигурации Autorun, в которых ошибка (или выбранные коды возврата запущенного сценария) отображается светодиодом после завершения. Очистить индикацию ошибки можно только с помощью команды сценария, выполняемой от имени root (например, через планирование задач).

Команда сценария для управления светодиодом выглядит, например, так: echo "7" > /dev/ttyS1

Вместо 7 в примере (для выключения светодиода состояния) следует использовать следующие символы:
Код Команда Действие
7 echo "7" > /dev/ttyS1 Светодиод состояния выключен
8 эхо "8" > /dev/ttyS1 Светодиод состояния зеленый
9 echo "9" > /dev/ttyS1 Светодиод состояния зеленый мигающий
: echo ":" > /dev/ttyS1 Светодиод состояния оранжевый
; эхо ";" > /dev/ttyS1 Светодиод состояния оранжевый мигающий
@ echo "@" > /dev/ttyS1 Светодиод копирования включен
A echo "A" > /dev/ttyS1 Светодиод копирования мигает
B echo "B" > /dev/ttyS1 Светодиод копирования выключен
