Примечания к сценарию:

Как основной сценарий, выполняющий требуемое действие после подключения внешнего устройства хранения данных, так и дополнительный вторичный сценарий, выполняемый после извлечения устройства хранения данных, требуют некоторых особых соображений:

  • Оба сценария запускаются демоном udev. Они выполняются с привилегиями root.
  • Среда сценариев практически пуста. В частности, путь к командам, например, из каталога /bin/, не задан как обычно! Т.е. путь должен быть указан либо с каждой командой, либо вы устанавливаете переменную окружения PATH в начале скрипта.
  • Переменная окружения SHELL указывает на /bin/ash. В DSM 7.x это ссылка на /bin/bash. Однако BASH может быть запущен в режиме, в котором, например, "подмена процесса" недоступна. Может случиться так, что скрипт, который прекрасно работает при интерактивном запуске, вдруг перестанет выполняться при запуске через демон udev из-за синтаксической ошибки. Я рекомендую использовать #!/bin/bash в качестве shebang.
  • Для отладки может быть полезно посмотреть вывод journalctl -f в окне оболочки при подключении внешнего устройства хранения данных. Также может быть полезен просмотр файла /var/log/bash_err.log.
