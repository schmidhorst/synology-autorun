セキュリティ

スクリプトの位置：

オートランパッケージで起動したスクリプトを保存する場所は、基本的に2種類あります：

  • 外部記憶装置：この方法で
    • スクリプトが準備された記憶装置だけが動作を開始
    します。
    • その他の接続可能なストレージデバイスはすべて無視されます。
    • 利用可能なセキュリティオプションが使用されていない場合、ディスクステーションにアクセスできる人は、準備されたUSBスティックを挿入することで任意のことを行うことができます。
  • DSMの内部で：
    • 記憶媒体が挿入されるたびに、スクリプトが起動されます。
    • スクリプトは、現在のストレージデバイスでアクションを実行するかどうかを確認する必要があります。
    • 利用可能なセキュリティオプションは、ほとんどの場合、必要ありません。

上記のプライマリスクリプトに加えて、ストレージデバイスが排出された後に実行されるセカンダリスクリプトをオプションで用意することも可能である。このセカンドスクリプトは、もちろんDSMの内部のみに配置することができます。排出されたメモリへのアクセスは、それが開始される時点ではもはや不可能である。

すべてのスクリプトはroot権限で実行されます。したがって、内部に配置されたスクリプトは、DSMのファイルシステムのほぼすべての場所に配置することができます。管理者権限のないユーザは、少なくとも書き込みはできないはずです！

セキュリティオプションです：

チェックサム（SHA256ハッシュ）をもとに、あらかじめ登録されたスクリプトに限定して実行することができます。その場合、他のスクリプトは実行されません。
2種類のアプローチがあります：

  1. オートランパッケージのインストール時の設定で、「1 first script allowed」という設定があり、インストール後の最初のスクリプトのチェックサムを判断して登録します。このスクリプトとその未修正のコピーのみが実行されます。初回インストール時、および必要に応じて再インストール時に、セキュリティは自動的に最初のスクリプトで武装します。スクリプトの修正が必要になった場合は、この「1つの最初のスクリプトを許可する」設定で、再度オートランパッケージの再インストールを実行してください。古い登録は削除され、新しいスクリプトのチェックサムが許可された唯一のスクリプトとして登録されます。
  2. パッケージのインストール時に、最初に「2 Add any hash」が選択されます。これで、異なるスクリプトを持つすべての外部メモリが次々と接続されることになります。そうすれば、必要なチェックサムを登録し、追加することができます。その後、再度インストールを実行し、値を「3 Only previously registered」に変更する必要があります。それ以降、セキュリティが有効になります。
手順 B は、A を以前に使用した場合にも使用でき、新しいスクリプトバリアントなどの追加のチェックサムを、すでに登録されているスクリプトバリアントと並行して追加する必要があります。
