Autorun

O pacote "Autorun" pode acionar uma ação quando um disco externo é conectado a uma estação Synology. Isto é freqüentemente muito útil para a cópia automatizada de dados. Pelo menos na versão 7.1 do DSM, tal opção de evento está faltando no agendador de tarefas. Há apenas "Startup" e "Shutdown" como eventos nas "Tarefas acionadas", mas nenhum "Armazenamento externo USB inserido" ou similar.

Com "ação" significa antes de tudo a execução de um script de shell BASH. O script pode então copiar arquivos entre a estação de disco e o disco usando comandos como rsync. Ou o script também pode, por exemplo, iniciar uma tarefa definida no agendador de tarefas.

Como uma ação adicional, o disco também pode ser ejetado após a conclusão da ação. Como isto significa que os dados no meio de dados externo não são mais diretamente acessíveis, isto proporciona um certo grau de proteção dos dados no meio de dados externo contra destruição, por exemplo, por meio de ransomware. No entanto, como o suporte de dados também pode ser reintegrado ao sistema através de software, ele não é uma proteção completa.

Uma proteção ainda melhor contra resgate pode ser obtida da seguinte maneira: Usar um suporte de dados externo com sua própria fonte de alimentação, como é comum em discos rígidos USB de 3,5 polegadas. A fonte de alimentação é operada através de uma tomada inteligente. Esta tomada inteligente é então desligada após o meio de dados ter sido ejetado. Contanto que o software de resgate não sobreponha esta implementação, o suporte de dados externo que é desconectado do sistema desta forma é protegido. Advertência: Enquanto o disco externo é montado para uma atualização, não há praticamente nenhuma proteção contra o ransomware que poderia ter entrado em seu sistema antes! Portanto, use sempre pelo menos dois discos alternadamente para backups!
