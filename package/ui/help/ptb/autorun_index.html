Autorun

O pacote "Autorun" pode acionar uma ação quando um disco rígido externo é conectado a uma Estação Synology. Isto é muitas vezes muito útil para a cópia automática de dados. Pelo menos na versão 7.1 do DSM, tal opção de evento está faltando no agendador de tarefas. Há apenas "Startup" e "Shutdown" como eventos nas "Tarefas acionadas", mas nenhum "Armazenamento externo USB inserido" ou similar.

Por "Ação" entende-se principalmente a execução de um roteiro de shell BASH. O script pode então copiar arquivos entre a estação de disco e o disco usando comandos como rsync. Ou o script também pode, por exemplo, iniciar uma tarefa definida no agendador de tarefas.

Como uma ação adicional, o disco também pode ser ejetado após a conclusão da ação. Como isto significa que os dados no disco externo não são mais diretamente acessíveis, isto proporciona alguma proteção dos dados no disco externo contra a destruição, por exemplo, por meio de um resgate. Entretanto, como o disco também pode ser reintegrado ao sistema via software, esta não é uma proteção completa.

Uma proteção ainda melhor contra os resgates pode ser obtida da seguinte maneira: Usar um meio de dados externo com sua própria fonte de alimentação, como é comum em discos rígidos USB de 3,5 polegadas. A fonte de alimentação é operada através de uma tomada inteligente. Esta tomada inteligente é desligada depois que o disco é ejetado. Contanto que o software de resgate não sobreponha esta implementação, o disco externo que é desconectado do sistema desta forma é protegido. Advertência: Enquanto o disco externo é montado para uma atualização, não há praticamente nenhuma proteção contra o ransomware que possa ter se infiltrado previamente em seu sistema! Portanto, use sempre pelo menos dois discos em rotação para backups!
