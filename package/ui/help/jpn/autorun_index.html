オートラン

Autorun」パッケージは、外付けハードドライブがSynology Stationに接続されたときにアクションをトリガーすることができます。これは、データの自動コピーに非常に便利です。少なくとも DSM バージョン 7.1 では、タスク スケジューラーにこのようなイベント オプションがありません。トリガーされたタスク」にはイベントとして「スタートアップ」と「シャットダウン」があるだけで、「外部USBストレージが挿入されました」などはありません。

アクション」とは、主にBASHシェルスクリプトの実行を意味します。このスクリプトは、rsyncなどのコマンドを使用して、ディスクステーションとディスクの間でファイルを往復コピーすることができます。また、スクリプトはタスクスケジューラーで定義されたタスクを起動することもできます。

追加アクションとして、アクション完了後にディスクを排出することも可能です。これにより、外付けディスクのデータに直接アクセスできなくなるため、ランサムウェアなどによる破壊から外付けディスクのデータをある程度保護することができる。ただし、ディスクはソフトウェアでシステムに再統合することも可能なため、完全な保護とは言えません。

ランサムウェアに対するさらに優れた保護は、次の方法で実現できます：3.5インチUSBハードディスクで一般的な、独自の電源を持つ外部データ媒体を使用する。この電源は、スマートソケットを介して操作されます。そして、このスマートソケットは、ディスクが排出された後にオフにされます。ランサムウェアがこの実装を無効にしない限り、この方法でシステムから切り離された外付けディスクは保護されます。警告 外付けディスクがアップデートのためにマウントされている間は、以前にシステムに侵入した可能性のあるランサムウェアに対する保護は事実上ありません！したがって、バックアップには、常に少なくとも2つのディスクをローテーションで使用してください！
