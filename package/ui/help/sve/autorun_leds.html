LED-indikator för status och kopiering

Autorun kan styra status- eller kopierings-LED:n. Detta kan användas för att indikera en pågående aktivitet och vid behov ett fel.

Anmärkningar:

  • Det finns inget enkelt sätt att läsa av det aktuella LED-tillståndet eller återgå till det tidigare tillståndet. Om LED-kontrollen är aktiverad i Autoruns inställningar kommer det aktuella LED-tillståndet (t.ex. status som blinkar orange på grund av ett fel) eventuellt att skrivas över av Autorun. Och även vice versa kan andra händelser skriva över den display som ställts in av Autorun.
  • Det finns konfigurationsinställningar för Autorun där ett fel (eller valbara returkoder för det startade skriptet) indikeras av en lysdiod efter slutet. Det är då endast möjligt att radera felindikeringen med ett skriptkommando som utförs som root (t.ex. via uppgiftsschemaläggning).

Skriptkommandot för LED-kontrollen är t.ex. echo "7" > /dev/ttyS1

I stället för 7 i exemplet (för status-LED av) ska följande tecken användas:
Kod Kommando Åtgärd
7 echo "7" > /dev/ttyS1 Status-LED avstängd
8 echo "8" > /dev/ttyS1 Status-LED grön
9 echo "9" > /dev/ttyS1 Status-LED grön blinkar
: echo ":" > /dev/ttyS1 Status-LED orange
; echo ";" > /dev/ttyS1 Status-LED orange blinkar
@ echo "@" > /dev/ttyS1 Lysdioden för kopiering är på
A echo "A" > /dev/ttyS1 Kopierings-LED blinkar
B echo "B" > /dev/ttyS1 Kopiera lysdioden är släckt
