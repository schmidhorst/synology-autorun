安全性

脚本位置：

有两个根本不同的地方可以放置由自动运行包启动的脚本：

  • 在外部存储设备上：
    • 只有以这种方式为脚本准备的存储设备才会启动该动作。所有其他可能被插入的存储设备将被忽略。
    • 如果不使用可用的安全选项，那么任何能够访问磁盘站的人都可以通过插入准备好的U盘来做任意的事情。 例如，删除整个磁盘站并自由访问整个网络。
  • DSM中的内部：
    • 每次插入存储设备，脚本就会启动。脚本本身可能要检查是否应该在当前的存储设备上执行其动作。
    • 可用的安全选项在大多数情况下是没有必要的。

除了上面提到的主脚本外，还可以选择一个次要的脚本，在存储设备被弹出后执行，也是可以的。 这第二个脚本自然只能位于DSM的内部。在要启动的时候，对弹出的存储器的访问就不再可能了。

所有的脚本都以root权限运行。因此内部放置的脚本几乎可以位于DSM文件系统的任何地方。 没有管理员权限的用户不应该有访问权限，至少不应该有写入权限！！

安全选项：

脚本的执行可以通过检查和（SHA256哈希值）限制在以前注册的脚本上。其他脚本将不会被执行。
有两种不同的方法：

  1. 在自动运行包安装时的设置中，选择 "允许1个第一个脚本 "的设置，安装后的第一个脚本的校验和被确定并注册。 只有这个脚本和它的不变的副本被执行。安全性在初次安装后会自动与第一个脚本绑定，如果有必要，在重新安装后也会如此。 如果脚本需要改变，那么之后再进行自动运行包的重新安装，设置为 "允许1个第一个脚本"。旧的注册将被删除，新的脚本校验和将被注册为唯一允许的。
  2. 在软件包安装过程中，首先选择 "2 添加任何散列"。现在，所有有不同脚本的外部存储器都需要一个接一个地插入。这样，所需的校验和就可以被注册和添加。在这之后，需要再次运行安装程序，并将数值改为 "3 只有先前注册的"。从那时起，安全系统就开始启动了。
如果之前使用的是A，并且需要在之前注册的脚本变量的基础上添加额外的校验和，例如新的脚本变量，那么也可以使用程序B。
