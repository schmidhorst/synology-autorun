Segurança

Localização do guião:

Existem dois lugares fundamentalmente diferentes onde o guião iniciado pelo pacote autorun pode ser colocado:

  • Nos dispositivos de armazenamento externo:
    • Só os dispositivos de armazenamento preparados com o guião desta forma é que iniciarão a acção.
    • Todos os outros dispositivos de armazenamento que possam ser ligados serão ignorados.
    • Se a opção de segurança disponível não for utilizada, então qualquer pessoa que tenha acesso à estação de disco pode fazer coisas arbitrárias ligando uma pen USB preparada.
    • Por exemplo,
    • apagar toda a estação de disco e ter acesso livre a toda a rede.
  • Interno no DSM:
    • Sempre que um dispositivo de armazenamento é ligado, o script é iniciado
    .
    • O script em si pode ter de verificar se deve executar as suas acções no dispositivo de armazenamento actual.
    • A opção de segurança disponível não será necessária a maior parte
do tempo.

Para além do script primário acima mencionado, um script secundário, que é executado após o dispositivo de armazenamento ter sido ejectado, é também opcionalmente possível. Este segundo script, naturalmente, só pode ser localizado internamente no DSM. O acesso à memória ejectada já não é possível no momento em que esta deve ser iniciada.

Todos os guiões são executados com permissão da raiz. Os scripts colocados internamente podem, portanto, ser localizados em quase qualquer parte do sistema de ficheiros do DSM. Os utilizadores sem direitos de administrador não devem ter acesso, pelo menos não devem ter acesso de escrita!

Opção de segurança:

A execução de scripts pode ser restringida a scripts previamente registados usando o seu checksum (SHA256 hash). Outros roteiros não serão executados.
Há duas abordagens diferentes:

  1. Nas configurações durante a instalação do pacote autorun, a configuração "1 primeiro script permitido" é seleccionada, e o checksum do primeiro script após a instalação é determinado e registado. Apenas este script e cópias inalteradas do mesmo são executados. A segurança arma automaticamente com o primeiro script após a inicial, e se necessário também após uma reinstalação. Se o script alguma vez precisar de uma alteração, então, depois, fazer de novo uma reinstalação do pacote autorun com esta definição "1 primeiro script permitido". O antigo registo é apagado e o novo checksum do script é registado posteriormente como o único registo admissível.
  2. Durante a instalação do pacote primeiro "2 Adicionar qualquer hash" é seleccionado. Agora todos os armazéns externos que têm diferentes scripts precisam de ser ligados um após o outro. Para que os checksums necessários possam ser registados e adicionados. Depois disso, a instalação precisa de ser executada novamente e o valor tem de ser alterado para "3 Apenas previamente registado". A partir daí, a segurança está armada.
O procedimento B também pode ser utilizado se A estava anteriormente em uso e um checksum adicional, por exemplo, para uma nova variante de script, precisa de ser adicionado em paralelo com a variante de script previamente registada.
