Indicador LED de status e cópia

A Autorun pode controlar o status ou copiar o LED. Isto pode ser usado para indicar uma atividade em andamento e, se necessário, um erro.

Notas:

  • Não há uma maneira fácil de ler o estado atual dos LEDs ou retornar ao estado anterior. Se o controle LED estiver habilitado nas configurações do Autorun, então o estado atual do LED (por exemplo, estado piscando laranja devido a um erro) será possivelmente sobrescrito pelo Autorun. E também vice-versa, outros eventos podem sobrescrever o display definido pela Autorun.
  • Há definições de configuração da Autorun onde um erro (ou códigos de retorno selecionáveis do script iniciado) é indicado por um LED após o final. A limpeza da exibição do erro só é então possível com um comando de script executado como raiz (por exemplo, via agendamento de tarefas).

O comando do script para o controle LED é por exemplo echo "7" > /dev/ttyS1

Em vez dos 7 do exemplo (para status LED desligado), os seguintes caracteres devem ser usados:
Código Comando Ação
7 echo "7" > /dev/ttyS1 LED de status apagado
8 echo "8" > /dev/ttyS1 Status LED verde
9 echo "9" > /dev/ttyS1 Status LED verde piscando
: echo ":" > /dev/ttyS1 Estado LED laranja
; echo ";" > /dev/ttyS1 Estado LED laranja piscando
@ echo "@" > /dev/ttyS1 Copiar LED ligado
A echo "A" > /dev/ttyS1 Cópia LED piscando
B echo "B" > /dev/ttyS1 Copiar LED desligado
