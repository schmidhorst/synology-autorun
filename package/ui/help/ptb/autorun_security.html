Segurança

Localização do roteiro:

Há dois lugares fundamentalmente diferentes onde o roteiro iniciado pelo pacote autorun pode ser colocado:

  • Nos dispositivos de armazenamento externo:
    • Somente os dispositivos de armazenamento preparados com o script desta forma iniciarão a ação.
    • Todos os outros dispositivos de armazenamento que podem ser conectados serão ignorados.
    • Se a opção de segurança disponível não for utilizada, então qualquer um que tenha acesso à estação de disco pode fazer coisas arbitrárias conectando um dispositivo USB preparado.
    • Por exemplo,
    • apagar toda a estação de disco e ter acesso livre a toda a rede.
  • Interno no DSM:
    • Toda vez que um dispositivo de armazenamento é conectado, o script é iniciado
    .
    • O script em si pode ter que verificar se ele deve executar suas ações no dispositivo de armazenamento atual.
    • A opção de segurança disponível não será necessária na maior parte
do tempo.

Além do script primário mencionado acima, um script secundário, que é executado após o dispositivo de armazenamento ter sido ejetado, também é opcionalmente possível. Este segundo script, naturalmente, só pode ser localizado internamente no DSM. O acesso à memória ejetada não é mais possível no momento em que ela deve ser iniciada.

Todos os scripts são executados com permissão da raiz. Portanto, os scripts colocados internamente podem ser localizados em quase qualquer lugar no sistema de arquivos DSM. Usuários sem direitos de administrador não devem ter acesso, pelo menos não devem ter acesso de escrita!

Opção de segurança:

A execução de scripts pode ser restrita a scripts previamente registrados usando seu checksum (SHA256 hash). Outros roteiros não serão executados.
Há duas abordagens diferentes:

  1. Nas configurações durante a instalação do pacote autorun, a configuração "1 primeiro script permitido" é selecionada, e o checksum do primeiro script após a instalação é determinado e registrado. Somente este script e cópias inalteradas do mesmo são executados. A segurança arma automaticamente com o primeiro script após a instalação inicial, e se necessário também após uma reinstalação. Se o script precisar de uma mudança, então, depois, faça novamente uma reinstalação do pacote autorun com esta configuração "1 primeiro script permitido". O antigo registro é excluído e o novo checksum do script é registrado posteriormente como o único permitido.
  2. Durante a instalação do pacote primeiro "2 Adicionar qualquer hash" é selecionado. Agora todos os armazéns externos que têm diferentes scripts precisam ser conectados um após o outro. Para que os checksums necessários possam ser registrados e adicionados. Depois disso, a instalação precisa ser executada novamente e o valor tem que ser alterado para "3 Somente previamente registrado". A partir de então, a segurança está armada.
O procedimento B também pode ser usado se A estava em uso anteriormente e um checksum adicional, por exemplo, para uma nova variante de script, precisa ser adicionado em paralelo com a variante de script previamente registrada.
