Sicurezza

Posizione dello script:

Esistono due luoghi fondamentalmente diversi in cui può essere collocato lo script avviato dal pacchetto autorun:

  • Sui dispositivi di archiviazione esterni:
    • Solo i dispositivi di archiviazione preparati con lo script in questo modo avvieranno l'azione.
    • Se non si utilizza l'opzione di sicurezza disponibile, chiunque abbia accesso alla stazione disco può fare cose arbitrarie inserendo una chiavetta USB preparata, ad esempio cancellare l'intera stazione disco e avere libero accesso all'intera rete.
  • interno del DSM:
    • ogni volta che viene inserito un dispositivo di archiviazione, viene avviato lo script.
Lo
    • script stesso potrebbe dover verificare se deve eseguire le sue azioni sul dispositivo di archiviazione corrente.
    • L'opzione di sicurezza disponibile non sarà necessaria nella maggior parte dei casi.

Oltre allo script principale menzionato in precedenza, è anche possibile creare uno script secondario, che viene eseguito dopo l'espulsione del dispositivo di archiviazione. Questo secondo script può naturalmente trovarsi solo all'interno del DSM. L'accesso alla memoria espulsa non è più possibile al momento dell'avvio.

Tutti gli script vengono eseguiti con i permessi di root. Gli script collocati internamente possono quindi essere collocati quasi ovunque nel file system del DSM. Gli utenti senza diritti di amministratore non dovrebbero avere accesso, almeno non in scrittura!

Opzione di sicurezza:

L'esecuzione degli script può essere limitata agli script precedentemente registrati utilizzando il loro checksum (hash SHA256). Gli altri script non verranno eseguiti.
Esistono due approcci diversi:

  1. Nelle impostazioni durante l'installazione del pacchetto autorun, viene selezionata l'impostazione "1 primo script consentito" e viene determinato e registrato il checksum del primo script dopo l'installazione. Solo questo script e le sue copie invariate vengono eseguite. La sicurezza si arma automaticamente con il primo script dopo l'installazione iniziale e, se necessario, anche dopo una reinstallazione. Se dovesse essere necessario modificare lo script, è necessario eseguire nuovamente una reinstallazione del pacchetto di esecuzione automatica con l'impostazione "1 primo script consentito". La vecchia registrazione viene cancellata e il nuovo checksum dello script viene registrato in seguito come l'unico ammissibile.
  2. Durante l'installazione del pacchetto viene selezionato per primo "2 Aggiungi qualsiasi hash". Ora tutti gli archivi esterni che hanno script diversi devono essere collegati uno dopo l'altro. In questo modo è possibile registrare e aggiungere i checksum necessari. Dopodiché è necessario eseguire di nuovo l'installazione e cambiare il valore in "3 Only previously registered". Da quel momento in poi la sicurezza è attiva.
La procedura B può essere utilizzata anche nel caso in cui la procedura A sia stata utilizzata in precedenza e sia necessario aggiungere un checksum aggiuntivo, ad esempio per una nuova variante di script, in parallelo alla variante di script registrata in precedenza.
