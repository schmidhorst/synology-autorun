Autorun

Het pakket "Autorun" kan een actie activeren wanneer een externe harde schijf wordt aangesloten op een Synology Station. Dit is vaak zeer nuttig voor het automatisch kopiëren van gegevens. Tenminste in DSM-versie 7.1 ontbreekt een dergelijke gebeurtenisoptie in de taakplanner. Er zijn alleen "Opstarten" en "Afsluiten" als gebeurtenissen in de "Geactiveerde taken", maar geen "Externe USB-opslag geplaatst" of iets dergelijks.

Met "Actie" wordt vooral het uitvoeren van een BASH-shell-script bedoeld. Het script kan dan bestanden heen en weer kopiëren tussen het schijfstation en de schijf met behulp van commando's als rsync. Of het script kan bijvoorbeeld ook een in de taakplanner gedefinieerde taak starten.

Als extra actie kan de schijf ook worden uitgeworpen nadat de actie is voltooid. Aangezien dit betekent dat de gegevens op de externe schijf niet langer rechtstreeks toegankelijk zijn, biedt dit enige bescherming van de gegevens op de externe schijf tegen vernietiging, bijvoorbeeld door ransomware. Aangezien de schijf echter ook via software opnieuw in het systeem kan worden geïntegreerd, is dit geen volledige bescherming.

Een nog betere bescherming tegen ransomware kan op de volgende manier worden bereikt: Gebruik een extern gegevensmedium met een eigen voeding, zoals gebruikelijk bij 3,5 inch USB harde schijven. De voeding wordt bediend via een slim stopcontact. Dit slimme stopcontact wordt dan uitgeschakeld nadat de schijf is uitgeworpen. Zolang ransomware deze implementatie niet overruled, is de externe schijf die op deze manier van het systeem wordt losgekoppeld beschermd. Waarschuwing: Terwijl de externe schijf is aangekoppeld voor een update, is er vrijwel geen bescherming tegen ransomware die mogelijk eerder uw systeem heeft geïnfiltreerd! Gebruik daarom altijd minstens twee schijven bij toerbeurt voor back-ups!
