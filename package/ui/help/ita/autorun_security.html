Sicurezza

Posizione dello script:

Esistono due posizioni fondamentalmente diverse in cui può essere memorizzato lo script avviato dal pacchetto autorun:

  • Sui dispositivi di archiviazione esterni:
    • solo i dispositivi di archiviazione preparati con lo script in questo modo avvieranno l'azione.
    • Tutti gli altri dispositivi di archiviazione collegabili vengono ignorati.
    • Se non si utilizza l'opzione di sicurezza disponibile, chiunque abbia accesso alla stazione disco può fare cose arbitrarie inserendo una chiavetta USB preparata, ad esempio cancellare l'intera stazione disco e avere libero accesso all'intera rete.
  • Internamente al DSM:
    • Ogni volta che viene inserito un supporto di memorizzazione, viene avviato lo script.
    • Lo script stesso potrebbe dover verificare se deve eseguire le sue azioni sul dispositivo di archiviazione corrente.
    • L'opzione di sicurezza disponibile non sarà necessaria nella maggior parte dei casi.

Oltre allo script primario sopra menzionato, è possibile avere uno script secondario opzionale che viene eseguito dopo l'espulsione del dispositivo di memorizzazione. Questo secondo script può, ovviamente, trovarsi solo all'interno del DSM. L'accesso alla memoria espulsa non è più possibile al momento dell'avvio.

Tutti gli script vengono eseguiti con i privilegi di root. Gli script posizionati internamente possono quindi trovarsi quasi ovunque nel file system del DSM. Gli utenti senza diritti di amministratore non dovrebbero avere accesso, almeno non in scrittura!

Opzione di sicurezza:

L'esecuzione degli script può essere limitata agli script precedentemente registrati in base al loro checksum (hash SHA256). Gli altri script non verranno eseguiti.
Esistono due approcci diversi:

  1. Nelle impostazioni al momento dell'installazione del pacchetto autorun, viene selezionata l'impostazione "1 primo script consentito" e viene determinata e registrata la checksum del primo script dopo l'installazione. Solo questo script e le sue copie non modificate vengono eseguite. La sicurezza si arma automaticamente con il primo script dopo la prima installazione e, se necessario, dopo la reinstallazione. Se lo script deve essere modificato, eseguire un'altra reinstallazione del pacchetto di esecuzione automatica con l'impostazione "1 primo script consentito". La vecchia registrazione viene eliminata e il nuovo checksum dello script viene successivamente registrato come l'unico consentito.
  2. Durante l'installazione del pacchetto, viene selezionato prima "2 Aggiungi qualsiasi hash". A questo punto tutte le memorie esterne, che hanno script diversi, devono essere collegate una dopo l'altra. In questo modo è possibile registrare e aggiungere i checksum richiesti. Successivamente, l'installazione deve essere eseguita di nuovo e il valore deve essere modificato in "3 Only previously registered". Da quel momento in poi, la sicurezza è abilitata.
La procedura B può essere utilizzata anche se in precedenza è stata utilizzata la procedura A e se è necessario aggiungere un checksum aggiuntivo, ad esempio per una nuova variante di script, in parallelo alla variante di script già registrata.
