Wskaźnik LED stanu i kopiowania

Program Autorun może sterować diodą LED stanu lub kopiowania. Można tego użyć do wskazania trwającej czynności i, w razie potrzeby, błędu.

Uwagi:

  • Nie ma prostego sposobu na odczytanie aktualnego stanu diody LED lub powrót do poprzedniego stanu. Jeżeli w ustawieniach programu Autorun włączono sterowanie diodami LED, wówczas aktualny stan diod (np. status migający na pomarańczowo z powodu błędu) zostanie ewentualnie nadpisany przez program Autorun. Również odwrotnie inne zdarzenia mogą nadpisać wyświetlacz ustawiony przez program Autorun.
  • Istnieją ustawienia konfiguracyjne programu Autorun, w których błąd (lub możliwe do wyboru kody powrotne uruchomionego skryptu) jest sygnalizowany diodą LED po zakończeniu. Wyczyszczenie wskazania błędu jest wtedy możliwe tylko za pomocą polecenia skryptu wykonywanego jako root (np. poprzez planowanie zadań).

Polecenie skryptowe do sterowania diodą LED jest np. echo "7" > /dev/ttyS1

Zamiast 7 w przykładzie (dla wyłączenia diody LED) należy użyć następujących znaków:
Kod Polecenie Działanie
7 echo "7" > /dev/ttyS1 Dioda statusu wyłączona
8 echo "8" > /dev/ttyS1 Dioda statusu zielona
9 echo "9" > /dev/ttyS1 Dioda statusu zielona miga
: echo ":" > /dev/ttyS1 Dioda LED stanu pomarańczowa
; echo ";" > /dev/ttyS1 Dioda LED statusu pomarańczowa migająca
@ echo "@" > /dev/ttyS1 Dioda LED kopiowania włączona
A echo "A" > /dev/ttyS1 Migająca dioda kopiowania
B echo "B" > /dev/ttyS1 Dioda LED kopiowania wyłączona
