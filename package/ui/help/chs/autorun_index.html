自动运行

当外部硬盘连接到Synology Station时，"Autorun "软件包可以触发一个动作。这对于自动复制数据通常非常有用。至少在DSM 7.1版本中，任务调度器中缺少这样一个事件选项。在 "触发的任务 "中只有 "启动 "和 "关闭 "这两个事件，而没有 "插入外部USB存储器 "或类似事件。

所谓 "行动 "主要是指执行一个BASH shell脚本。该脚本可以使用rsync等命令在磁盘站和磁盘之间来回复制文件。或者该脚本也可以启动一个在任务调度器中定义的任务。

作为一个额外的动作，磁盘也可以在动作完成后被弹出。因为这意味着外部磁盘上的数据不再可以直接访问，这为外部磁盘上的数据提供了一些保护，使其免受破坏，例如被勒索软件破坏。然而，由于磁盘也可以通过软件重新整合到系统中，这并不是完全的保护。

甚至可以通过以下方式实现对勒索软件更好的保护：使用有自己电源的外部数据介质，如3.5英寸USB硬盘的常见做法。该电源是通过一个智能插座操作的。然后在磁盘被弹出后关闭这个智能插座。只要勒索软件不覆盖这一实施，以这种方式从系统中断开的外部磁盘就会受到保护。警告： 当外部磁盘被挂载以进行更新时，几乎没有任何保护措施可以抵御可能先前已经渗透到你的系统中的勒索软件！因此，一定要使用至少两个磁盘！因此，请始终使用至少两个磁盘轮流进行备份
