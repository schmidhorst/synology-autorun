Sécurité

Emplacement du script :

Il existe deux endroits fondamentalement différents où le script lancé par le paquet d'exécution automatique peut être placé :

  • Sur les périphériques de stockage externes :
    • Seuls les périphériques de stockage préparés de cette manière avec le script lanceront l'action.
    • Si l'option de sécurité disponible n' est pas utilisée, toute personne ayant accès à la station de disque peut faire des choses arbitraires en branchant une clé USB préparée, par exemple supprimer toute la station de disque et avoir un accès libre à l'ensemble du réseau.

Outre le script principal mentionné ci-dessus, un script secondaire, qui est exécuté après l'éjection du périphérique de stockage, est également possible en option. Ce second script ne peut naturellement être situé qu'en interne sur le DSM. L'accès à la mémoire éjectée n'est plus possible au moment où elle doit être démarrée.

Tous les scripts sont exécutés avec l'autorisation de l'administrateur. Les scripts placés en interne peuvent donc se trouver presque n'importe où dans le système de fichiers du DSM. Les utilisateurs sans droits d'administrateur ne devraient pas y avoir accès, du moins pas en écriture !

Option de sécurité :

L'exécution des scripts peut être limitée aux scripts précédemment enregistrés à l'aide de leur somme de contrôle (hachage SHA256). Les autres scripts ne seront pas exécutés.
Il existe deux approches différentes :

  1. Lors de l'installation du paquet autorun, le paramètre "1 premier script autorisé" est sélectionné, et la somme de contrôle du premier script après l'installation est déterminée et enregistrée. Seul ce script et ses copies inchangées sont exécutés. La sécurité s'arme automatiquement avec le premier script après l'installation initiale et, si nécessaire, après une réinstallation. Si le script doit être modifié, il faut ensuite procéder à une réinstallation automatique du paquet avec le paramètre "1 premier script autorisé". L'ancien enregistrement est supprimé et la nouvelle somme de contrôle du script est enregistrée par la suite comme étant la seule autorisée.
  2. Lors de l'installation du paquet, le premier paramètre "2 Add any hash" (Ajouter n'importe quel hachage) est sélectionné. Maintenant, toutes les unités de stockage externes qui ont des scripts différents doivent être branchées l'une après l'autre. Ainsi, les sommes de contrôle nécessaires peuvent être enregistrées et ajoutées. Ensuite, l'installation doit être exécutée à nouveau et la valeur doit être changée en "3 Only previously registered". Dès lors, la sécurité est activée.
La procédure B peut également être utilisée si la procédure A a été utilisée précédemment et qu'une somme de contrôle supplémentaire, par exemple pour une nouvelle variante de script, doit être ajoutée parallèlement à la variante de script précédemment enregistrée.
