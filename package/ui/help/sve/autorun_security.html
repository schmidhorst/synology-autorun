Säkerhet

Skriptets plats:

Det finns två principiellt olika platser där skriptet som startas av autorun-paketet kan lagras:

  • På externa lagringsenheter:
    • Endast lagringsenheter som förberetts med skriptet på detta sätt startar åtgärden .
    • Alla andra lagringsenheter som kan anslutas ignoreras.
    • Om det tillgängliga säkerhetsalternativet inte används kan vem som helst som har tillgång till diskstationen göra godtyckliga saker genom att sätta in ett förberett USB-minne, t.ex. radera hela diskstationen och få fri tillgång till hela nätverket.
  • Internt i DSM:
    • Varje gång ett lagringsmedium sätts in startas skriptet .
    • Själva skriptet kan behöva kontrollera om det ska utföra sina åtgärder på det aktuella lagringsmediet.
    • Det tillgängliga säkerhetsalternativet kommer inte att behövas i de flesta fall.

Förutom det primära skriptet som nämns ovan är det också möjligt att ha ett valfritt sekundärt skript som körs efter att lagringsmediet har kastats ut. Detta andra skript kan naturligtvis endast finnas internt på DSM. Tillgång till det utkastade minnet är inte längre möjlig vid den tidpunkt då det ska startas.

Alla skript körs med rooträttigheter. Internt placerade skript kan därför placeras nästan var som helst i DSM:s filsystem. Användare utan administratörsrättigheter bör inte ha åtkomst, åtminstone inte skrivåtkomst!

Säkerhetsalternativ:

Exekvering av skript kan begränsas till tidigare registrerade skript baserat på deras kontrollsumma (SHA256-hash). Andra skript kommer då inte att exekveras.
Det finns två olika tillvägagångssätt:

  1. I inställningarna vid installation av autorun-paketet väljs inställningen "1 first script allowed", och kontrollsumman för det första skriptet efter installationen bestäms och registreras. Endast detta skript och oförändrade kopior av det exekveras. Säkerheten rustar sig automatiskt med det första skriptet efter den första, och vid behov, ominstallationen. Om skriptet någonsin behöver ändras, utför då en ny ominstallation av autorun-paketet med denna inställning "1 första skript tillåtet". Den gamla registreringen raderas och det nya skriptets kontrollsumma registreras därefter som den enda tillåtna.
  2. Under paketinstallationen väljs först "2 Add any hash". Nu måste alla externa minnen, som har olika skript, anslutas efter varandra. Så att de nödvändiga kontrollsummorna kan registreras och läggas till. Därefter måste installationen utföras igen och värdet måste ändras till "3 Endast tidigare registrerade". Från och med då är säkerheten aktiverad.
Förfarande B kan också användas om A använts tidigare och en ytterligare kontrollsumma, t.ex. för en ny skriptvariant, måste läggas till parallellt med den redan registrerade skriptvarianten.
