Beveiliging

Locatie van het script:

Er zijn twee fundamenteel verschillende plaatsen waar het door het autorunpakket gestarte script kan worden geplaatst:

  • Op de externe opslagapparaten:
    • Alleen de opslagapparaten die op deze manier met het script zijn voorbereid, zullen de actie starten.
    • Alle andere opslagapparaten die kunnen worden ingeplugd, worden genegeerd
    • .
    • Als de beschikbare beveiligingsoptie niet wordt gebruikt, dan kan iedereen die toegang heeft tot het schijfstation willekeurige dingen doen door een geprepareerde USB-stick in te pluggen.
    • Bijvoorbeeld
    • het hele schijfstation wissen en vrije toegang hebben tot het hele netwerk.
  • Intern in DSM:
    • Telkens wanneer een opslagapparaat wordt ingeplugd, wordt het script gestart. Het is mogelijk dat het script zelf moet controleren of het zijn acties moet uitvoeren op het huidige opslagapparaat.
    • De beschikbare beveiligingsoptie zal meestal niet nodig zijn.

Naast het bovengenoemde primaire script is optioneel ook een secundair script mogelijk, dat wordt uitgevoerd nadat het opslagapparaat is uitgeworpen. Dit tweede script kan zich uiteraard alleen intern op de DSM bevinden. Toegang tot het uitgeworpen geheugen is niet meer mogelijk op het moment dat het moet worden gestart.

Alle scripts draaien met root-toestemming. Intern geplaatste scripts kunnen dus bijna overal in het DSM-bestandssysteem staan. Gebruikers zonder beheerdersrechten mogen geen toegang hebben, althans geen schrijftoegang!

Beveiligingsoptie:

Scriptuitvoering kan worden beperkt tot eerder geregistreerde scripts aan de hand van hun checksum (SHA256 hash). Andere scripts worden niet uitgevoerd.
Er zijn twee verschillende benaderingen:
