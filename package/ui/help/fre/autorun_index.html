Exécution automatique

Le paquet "Autorun" peut déclencher une action lorsqu'un disque dur externe est connecté à une station Synology. Ceci est souvent très utile pour la copie automatique de données. Au moins dans la version 7.1 du DSM, une telle option d'événement est absente dans le planificateur de tâches. Il n'y a que "Démarrage" et "Arrêt" comme événements dans les "Tâches déclenchées", mais pas de "Stockage USB externe inséré" ou similaire.

Par "action", on entend principalement l'exécution d'un script shell BASH. Le script peut ensuite copier des fichiers entre la station de disque et le disque à l'aide de commandes telles que rsync. Le script peut également lancer une tâche définie dans le planificateur de tâches.

Comme action supplémentaire, le disque peut également être éjecté une fois l'action terminée. Étant donné que les données du disque externe ne sont plus directement accessibles, elles sont protégées contre la destruction, par exemple par un ransomware. Toutefois, étant donné que le disque peut également être réintégré dans le système par le biais d'un logiciel, il ne s'agit pas d'une protection complète.

Une protection encore meilleure contre les ransomwares peut être obtenue de la manière suivante : Utiliser un support de données externe doté de sa propre alimentation, comme c'est souvent le cas pour les disques durs USB de 3,5 pouces. L'alimentation électrique est assurée par une prise intelligente. Cette prise intelligente est ensuite éteinte après l'éjection du disque. Tant que le ransomware n'annule pas cette mise en œuvre, le disque externe ainsi déconnecté du système est protégé. Attention : Pendant que le disque externe est monté pour une mise à jour, il n'y a pratiquement aucune protection contre les ransomwares qui ont pu s'infiltrer dans votre système ! Par conséquent, utilisez toujours au moins deux disques en rotation pour les sauvegardes !
