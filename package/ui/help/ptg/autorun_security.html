Segurança

Localização do guião:

Existem dois locais fundamentalmente diferentes onde o guião iniciado pelo pacote autorun pode ser armazenado:

  • Nos dispositivos de armazenamento externo:
    • apenas os dispositivos de armazenamento preparados com o guião desta forma iniciarão a acção .
    • Todos os outros dispositivos de armazenamento que podem ser ligados à corrente são ignorados.
    • Se a opção de segurança disponível não for utilizada, qualquer pessoa que tenha acesso à estação de disco pode fazer coisas arbitrárias inserindo uma pen USB preparada, por exemplo, limpar toda a estação de disco e ter acesso livre a toda a rede.
  • Internamente em DSM:
    • Cada vez que um suporte de armazenamento é inserido, o script é iniciado .
    • O próprio script pode precisar de verificar se deve executar as suas acções no dispositivo de armazenamento actual.
    • A opção de segurança disponível não será necessária na maioria dos casos.

Para além do script primário acima mencionado, é também possível ter um script secundário opcional que é executado após o dispositivo de armazenamento ter sido ejectado. Este segundo guião só pode, evidentemente, ser localizado internamente no DSM. O acesso à memória ejectada já não é possível no momento em que esta deve ser iniciada.

Todos os guiões são executados com privilégios de raiz. Os scripts colocados internamente podem, portanto, ser localizados em quase qualquer parte do sistema de ficheiros do DSM. Os utilizadores sem direitos de administrador não devem ter acesso, pelo menos não devem ter acesso de escrita!

Opção de segurança:

A execução de scripts pode ser restrita a scripts previamente registados com base no seu checksum (SHA256 hash). Outros roteiros não serão então executados.
Há duas abordagens diferentes:

  1. Nas definições ao instalar o pacote autorun, a definição "1 primeiro script permitido" é seleccionada, e o checksum do primeiro script após a instalação é determinado e registado. Apenas este script e cópias não modificadas do mesmo são executadas. A segurança arma-se automaticamente com o primeiro script após a instalação inicial, e se necessário, reinstalação. Se o script necessitar de modificação, então executar outra reinstalação do pacote autorun com esta configuração "1 primeiro script permitido". O antigo registo é apagado e o novo checksum do script é subsequentemente registado como o único permitido.
  2. Durante a instalação do pacote, "2 Adicionar qualquer hash" é seleccionado primeiro. Agora todas as memórias externas, que têm diferentes scripts, devem ser ligadas uma após a outra. Para que os checksums necessários possam ser registados e adicionados. Depois disso, a instalação tem de ser executada novamente e o valor tem de ser alterado para "3 Apenas previamente registado". A partir daí, a segurança é activada.
O procedimento B também pode ser utilizado se A foi utilizado antes e um checksum adicional, por exemplo, para uma nova variante de script, deve ser adicionado em paralelo com a variante de script já registada.
