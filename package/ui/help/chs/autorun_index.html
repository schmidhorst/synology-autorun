自动运行

当外部磁盘插入Synology工作站时，"Autorun "软件包可以触发一个动作。这对于自动复制数据通常非常有用。 至少在DSM7.1版本中，任务调度器中缺少这样一个事件选项。 在 "触发的任务 "中只有 "启动 "和 "关闭 "这两个事件，而没有 "插入外部USB存储器 "或类似的事件。

行动 "首先是指执行一个BASH shell脚本。 该脚本可以使用rsync等命令在磁盘站和磁盘之间来回复制文件。 或者脚本也可以启动一个在任务调度器中定义的任务。

作为一个额外的动作，磁盘也可以在动作完成后被弹出。 因为这意味着外部数据介质上的数据不再可以直接访问，这为外部数据介质上的数据提供了一定程度的保护，防止被破坏，例如被勒索软件破坏。 然而，由于数据介质也可以通过软件重新整合到系统中，所以这并不是完全的保护。

甚至更好的勒索软件保护可以通过以下方式实现：使用有自己的电源的外部数据介质，如3.5英寸USB硬盘的常见做法。该电源通过一个智能插座操作。在数据介质被弹出后，这个智能插座就会被关闭。只要勒索软件不覆盖这一实施，以这种方式与系统断开连接的外部数据介质就会受到保护。 警告： 当外部磁盘被安装以进行更新时，实际上没有任何保护措施来防止之前可能已经进入你的系统的勒索软件！因此，总是使用至少两个磁盘来进行更新！ 因此，请至少交替使用两个磁盘进行备份！
