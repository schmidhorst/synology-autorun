Segurança

Localização do roteiro:

Há dois locais fundamentalmente diferentes onde o roteiro iniciado pelo pacote autorun pode ser armazenado:

  • Nos dispositivos de armazenamento externo:
    • somente os dispositivos de armazenamento preparados com o script desta forma iniciarão a ação .
    • Todos os outros dispositivos de armazenamento que podem ser conectados são ignorados.
    • Se a opção de segurança disponível não for utilizada, qualquer pessoa que tenha acesso à estação de disco pode fazer coisas arbitrárias inserindo um dispositivo USB preparado, por exemplo, limpar toda a estação de disco e ter acesso livre a toda a rede.
  • Internamente no DSM:
    • Toda vez que um meio de armazenamento é inserido, o script é iniciado .
    • O próprio script pode precisar verificar se ele deve executar suas ações no dispositivo de armazenamento atual.
    • A opção de segurança disponível não será necessária na maioria dos casos.

Além do script primário mencionado acima, também é possível ter um script secundário opcional que é executado após o dispositivo de armazenamento ter sido ejetado. Este segundo script só pode, é claro, ser localizado internamente no DSM. O acesso à memória ejetada não é mais possível no momento em que ela deve ser iniciada.

Todos os scripts são executados com privilégios de raiz. Portanto, os scripts colocados internamente podem ser localizados em quase qualquer lugar no sistema de arquivos do DSM. Usuários sem direitos de administrador não devem ter acesso, pelo menos não devem ter acesso de escrita!

Opção de segurança:

A execução de scripts pode ser restrita a scripts previamente registrados com base em seu checksum (SHA256 hash). Outros scripts não serão então executados.
Há duas abordagens diferentes:

  1. Nas configurações ao instalar o pacote autorun, a configuração "1 primeiro script permitido" é selecionada, e o checksum do primeiro script após a instalação é determinado e registrado. Somente este script e cópias não modificadas do mesmo são executadas. A segurança se arma automaticamente com o primeiro script após a instalação inicial e, se necessário, a reinstalação. Se o script precisar de modificação, então execute outra reinstalação do pacote autorun com esta configuração "1 primeiro script permitido". O antigo registro é excluído e o novo checksum do script é posteriormente registrado como o único permitido.
  2. Durante a instalação do pacote, "2 Adicionar qualquer hash" é selecionado primeiro. Agora todas as memórias externas, que têm diferentes scripts, devem ser conectadas uma após a outra. Para que os checksums necessários possam ser registrados e adicionados. Depois disso, a instalação tem que ser executada novamente e o valor tem que ser alterado para "3 Somente previamente registrado". A partir de então, a segurança é habilitada.
O procedimento B também pode ser usado se A foi usado antes e um checksum adicional, por exemplo, para uma nova variante de script, deve ser adicionado em paralelo com a variante de script já registrada.
