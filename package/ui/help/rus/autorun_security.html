Безопасность

Расположение скрипта:

Существует два принципиально разных места, где может быть размещен сценарий, запускаемый пакетом автозапуска:

  • На внешних устройствах хранения:
    • Только устройства хранения данных, подготовленные таким образом со сценарием, будут запускать действие.
    • Все остальные накопители, которые могут быть подключены, будут проигнорированы.
    • Если не использовать имеющуюся опцию безопасности, то любой, кто имеет доступ к дисковой станции, может делать произвольные вещи, подключив подготовленный USB-накопитель.

  • Внутренний в DSM:
    • Каждый раз, когда подключается накопитель, запускается скрипт. Сам скрипт может проверить, должен ли он выполнять свои действия на текущем устройстве хранения данных.
    • Доступная опция безопасности в большинстве случаев не потребуется.

В дополнение к основному скрипту, упомянутому выше, опционально возможен также вторичный скрипт, который выполняется после извлечения устройства хранения данных. Этот вторичный сценарий, естественно, может быть расположен только внутри DSM. Доступ к извлеченному запоминающему устройству в момент его запуска уже невозможен.

Все скрипты запускаются с правами root. Поэтому внутренние скрипты могут быть расположены практически в любом месте файловой системы DSM. Пользователи без прав администратора не должны иметь доступа, по крайней мере, доступа на запись!

Опция безопасности:

Выполнение скриптов может быть ограничено ранее зарегистрированными скриптами с помощью их контрольной суммы (хэш SHA256). Другие скрипты выполняться не будут.
Существует два различных подхода:

  1. В настройках при установке пакета автозапуска выбирается параметр "Разрешен 1 первый скрипт", определяется и регистрируется контрольная сумма первого скрипта после установки. Выполняется только этот скрипт и его неизмененные копии. Защита автоматически вооружается первым скриптом после первоначальной, а при необходимости и после повторной установки. Если сценарий необходимо изменить, то после этого снова произведите переустановку пакета автозапуска с настройкой "Разрешен 1 первый сценарий". Старая регистрация удаляется, а новая контрольная сумма скрипта регистрируется как единственно допустимая.
  2. Во время установки пакета сначала выбирается "2 Добавить любой хэш". Теперь все внешние хранилища с различными сценариями должны быть подключены одно за другим. Таким образом, необходимые контрольные суммы будут зарегистрированы и добавлены. После этого нужно снова запустить установку и изменить значение на "3 Только ранее зарегистрированные". После этого защита будет включена.
Процедура B также может быть использована, если ранее использовалась процедура A и необходимо добавить дополнительную контрольную сумму, например, для нового варианта сценария, параллельно с ранее зарегистрированным вариантом сценария.
