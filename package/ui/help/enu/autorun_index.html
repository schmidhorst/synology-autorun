Autorun

The "Autorun" package can trigger an action when an external hard drive is connected to a Synology Station. This is often very useful for automatic copying of data. At least in DSM version 7.1, such an event option is missing in the task scheduler. There are only "Startup" and "Shutdown" as events in the "Triggered Tasks", but no "External USB storage inserted" or similar.

By "Action" is primarily meant the execution of a BASH shell script. The script can then copy files back and forth between the disk station and the disk using commands like rsync. Or the script can e.g. also start a task defined in the task scheduler.

As an additional action, the disk can also be ejected after the action is completed. Since this means that the data on the external disk is no longer directly accessible, this provides some protection of the data on the external disk from destruction, e.g. by ransomware. However, since the disk can also be reintegrated into the system via software, this is not complete protection.

Even better protection against ransomware can be achieved in the following way: Use an external data medium with its own power supply, as is common with 3.5-inch USB hard drives. The power supply is operated via a smart socket. This smart socket is then turned off after the disk is ejected. As long as ransomware does not override this implementation, the external disk that is disconnected from the system in this way is protected. Warning: While the external disk is mounted for an update, there is virtually no protection against ransomware that may have previously infiltrated your system! Therefore, always use at least two disks in rotation for backups!
