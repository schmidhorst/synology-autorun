安全性

脚本的位置：

有两个根本不同的位置可以储存由自动运行包启动的脚本：

  • 在外部存储设备上：
    • 只有以这种方式准备了脚本的存储设备才会启动该动作。
    • 所有其他可以插入的存储设备都会被忽略。
    • 如果不使用可用的安全选项，任何能够访问磁盘站的人都可以通过插入准备好的U盘来做任意的事情，例如擦除整个磁盘站并自由访问整个网络。
  • 在DSM内部：
    • 每次插入一个存储介质，脚本就会被启动。
    • 脚本本身可能需要检查它是否应该在当前的存储设备上执行其行动。
    • 在大多数情况下，可用的安全选项将不需要。

除了上面提到的主脚本外，还可以有一个可选的第二脚本，在存储设备被弹出后执行。当然，这个第二脚本只能位于DSM的内部。在要启动的时候，对弹出的存储器的访问就不再可能了。

所有脚本都以root权限运行。因此，内部放置的脚本几乎可以位于DSM的文件系统中的任何地方。没有管理员权限的用户不应该有访问权，至少不应该有写入权!

安全选项：

脚本的执行可以根据它们的校验和（SHA256 哈希值）限制在以前注册的脚本上。其他脚本将不会被执行。
有两种不同的方法：

  1. 在安装自动运行包时的设置中，选择 "允许1个第一个脚本 "的设置，安装后第一个脚本的校验和被确定并注册。只有这个脚本和它的未修改的副本才会被执行。在初次安装后，如果有必要，重新安装后，安全系统会自动用第一个脚本来武装自己。如果该脚本需要修改，那么就用这个 "允许1个第一脚本 "的设置执行另一个自动运行包的重新安装。旧的注册被删除，新的脚本校验和随后被注册为唯一允许的。
  2. 在软件包安装过程中，首先选择 "2添加任何散列"。现在，所有具有不同脚本的外部存储器都必须一个接一个地插上。这样，所需的校验和就可以被注册和添加。此后，必须再次执行安装，并将数值改为 "3 只有先前注册的"。从那时起，安全就被启用了。
如果之前使用了A，也可以使用程序B，并且必须在已经注册的脚本变体上同时添加额外的校验和，例如新的脚本变体。
