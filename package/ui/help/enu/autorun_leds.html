Status and Copy LED indicator

Autorun can control the status or copy LED. This can be used to indicate an activity in progress and, if necessary, an error.

Notes:

  • There is no easy way to read the current LED state or return to the previous state. If LED control is enabled in Autorun's settings, then the current LED state (e.g. status flashing orange because of an error) will be possibly overwritten by Autorun. And also vice versa other events may overwrite the display set by Autorun.
  • There are Autorun configuration settings where an error (or selectable return codes of the started script) is indicated by an LED after the end. Clearing the error display is then only possible with a script command executed as root (e.g. via task scheduling).

The script command for the LED control is e.g. echo "7" > /dev/ttyS1

Instead of the 7 in the example (for status LED off) the following characters are to be used:
Code Command Action
7 echo "7" > /dev/ttyS1 Status LED off
8 echo "8" > /dev/ttyS1 Status LED green
9 echo "9" > /dev/ttyS1 Status LED green blinking
: echo ":" > /dev/ttyS1 Status LED orange
; echo ";" > /dev/ttyS1 Status LED orange blinking
@ echo "@" > /dev/ttyS1 Copy LED on
A echo "A" > /dev/ttyS1 Copy LED blinking
B echo "B" > /dev/ttyS1 Copy LED off
