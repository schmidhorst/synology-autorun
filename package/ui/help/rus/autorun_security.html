Безопасность

Расположение сценария:

Существует два принципиально разных места, где может храниться сценарий, запускаемый пакетом автозапуска:

  • На внешних устройствах хранения данных:
    • только устройства хранения данных, подготовленные со сценарием таким образом, запускают действие.
    • Все остальные подключаемые устройства хранения игнорируются.
    • Если доступная опция безопасности не используется, любой, кто имеет доступ к дисковой станции, может совершать произвольные действия, вставив подготовленный USB-накопитель, например, стереть всю дисковую станцию и получить свободный доступ ко всей сети.
  • Внутри DSM:
    • Каждый раз, когда вставляется носитель информации, запускается сценарий.
    • Самому сценарию может потребоваться проверить, должен ли он выполнять свои действия на текущем устройстве хранения данных.
    • В большинстве случаев доступная опция безопасности не потребуется.

В дополнение к основному скрипту, упомянутому выше, можно также иметь необязательный вторичный скрипт, который выполняется после извлечения устройства хранения данных. Этот вторичный сценарий, разумеется, может быть расположен только внутри DSM. Доступ к извлеченному запоминающему устройству в момент его запуска уже невозможен.

Все скрипты запускаются с правами root. Поэтому размещенные внутри DSM скрипты могут быть расположены практически в любом месте файловой системы DSM. Пользователи без прав администратора не должны иметь доступа, по крайней мере, доступа на запись!

Опция безопасности:

Выполнение скриптов может быть ограничено ранее зарегистрированными скриптами на основе их контрольной суммы (хэш SHA256). Другие скрипты в этом случае не будут выполняться.
Существует два различных подхода:

  1. В настройках при установке пакета автозапуска выбирается параметр "Разрешен 1 первый скрипт", определяется и регистрируется контрольная сумма первого скрипта после установки. Выполняется только этот скрипт и его немодифицированные копии. Security автоматически вооружается первым скриптом после первоначальной и, при необходимости, повторной установки. Если сценарий когда-либо потребует изменения, выполните еще одну переустановку пакета автозапуска с настройкой "Разрешен 1 первый сценарий". Старая регистрация удаляется, а контрольная сумма нового сценария впоследствии регистрируется как единственная разрешенная.
  2. Во время установки пакета сначала выбирается "2 Добавить любой хэш". Теперь все внешние памяти, которые имеют различные сценарии, должны быть подключены одна за другой. Таким образом, необходимые контрольные суммы могут быть зарегистрированы и добавлены. После этого нужно снова выполнить установку и изменить значение на "3 Только ранее зарегистрированные". С этого момента защита включена.
Процедура B также может быть использована, если ранее использовалась процедура A, и дополнительная контрольная сумма, например, для нового варианта сценария, должна быть добавлена параллельно с уже зарегистрированным вариантом сценария.
