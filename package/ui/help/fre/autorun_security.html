Sécurité

Emplacement du script :

Il existe deux emplacements fondamentalement différents où le script lancé par le paquet d'exécution automatique peut être stocké :

  • Sur les périphériques de stockage externes :
    • seuls les périphériques de stockage préparés de cette manière avec le script démarreront l'action.
    • Tous les autres périphériques de stockage pouvant être branchés sont ignorés.
    • Si l'option de sécurité disponible n' est pas utilisée, toute personne ayant accès à la station de disque peut faire des choses arbitraires en insérant une clé USB préparée, par exemple effacer toute la station de disque et avoir un accès libre à l'ensemble du réseau.
  • En interne dans le DSM :
    • Chaque fois qu'un support de stockage est inséré, le script est lancé.
    • Le script lui-même peut avoir besoin de vérifier s'il doit effectuer ses actions sur le périphérique de stockage actuel.
    • L'option de sécurité disponible ne sera pas nécessaire dans la plupart des cas.

Outre le script principal mentionné ci-dessus, il est également possible d'avoir un script secondaire facultatif qui est exécuté après l'éjection du support de stockage. Ce second script ne peut, bien entendu, être situé qu'en interne sur le DSM. L'accès à la mémoire éjectée n'est plus possible au moment où il doit être lancé.

Tous les scripts sont exécutés avec les privilèges de l'administrateur. Les scripts placés en interne peuvent donc se trouver presque n'importe où dans le système de fichiers du DSM. Les utilisateurs sans droits d'administrateur ne devraient pas y avoir accès, du moins pas en écriture !

Option de sécurité :

L'exécution des scripts peut être limitée aux scripts précédemment enregistrés sur la base de leur somme de contrôle (hachage SHA256). Les autres scripts ne seront alors pas exécutés.
Il existe deux approches différentes :

  1. Dans les paramètres d'installation du paquet autorun, le paramètre "1 premier script autorisé" est sélectionné, et la somme de contrôle du premier script après l'installation est déterminée et enregistrée. Seul ce script et ses copies non modifiées sont exécutés. La sécurité s'arme automatiquement du premier script après la première installation et, si nécessaire, la réinstallation. Si le script doit être modifié, procédez à une nouvelle réinstallation du paquet d'exécution automatique avec ce paramètre "1 premier script autorisé". L'ancien enregistrement est supprimé et la nouvelle somme de contrôle du script est ensuite enregistrée comme étant la seule autorisée.
  2. Lors de l'installation du logiciel, le paramètre "2 Add any hash" est sélectionné en premier. Toutes les mémoires externes, qui ont des scripts différents, doivent maintenant être branchées l'une après l'autre. Ainsi, les sommes de contrôle requises peuvent être enregistrées et ajoutées. Ensuite, l'installation doit être exécutée à nouveau et la valeur doit être changée en "3 Only previously registered". Dès lors, la sécurité est activée.
La procédure B peut également être utilisée si la procédure A a été utilisée auparavant et qu'une somme de contrôle supplémentaire, par exemple pour une nouvelle variante de script, doit être ajoutée parallèlement à la variante de script déjà enregistrée.
