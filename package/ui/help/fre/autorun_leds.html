Indicateur LED d'état et de copie

L'Autorun peut contrôler le voyant d'état ou de copie. Celui-ci peut être utilisé pour indiquer une activité en cours et, le cas échéant, une erreur.

Notes : L'autorun peut contrôler le voyant d'état ou de copie :

  • Il n'existe pas de moyen simple de lire l'état actuel de la LED ou de revenir à l'état précédent. Si le contrôle des DEL est activé dans les paramètres de l'Autorun, l'état actuel des DEL (par exemple, l'état clignotant en orange en raison d'une erreur) sera éventuellement écrasé par l'Autorun. Et vice versa, d'autres événements peuvent écraser l'affichage défini par l'Autorun.
  • Dans certains paramètres de configuration de l'Autorun, une erreur (ou des codes de retour sélectionnables du script lancé) est indiquée par une DEL après la fin. L'effacement de l'affichage de l'erreur n'est alors possible qu'avec une commande de script exécutée en tant que super-utilisateur (par exemple via la planification des tâches).

La commande de script pour le contrôle des DEL est par exemple echo "7" > /dev/ttyS1

À la place du 7 dans l'exemple (pour la LED d'état éteinte), les caractères suivants doivent être utilisés :
Code Commande Action
7 echo "7" > /dev/ttyS1 LED d'état éteinte
8 echo "8" > /dev/ttyS1 LED d'état verte
9 echo "9" > /dev/ttyS1 LED d'état verte clignotante
: echo " :" > /dev/ttyS1 LED d'état orange
; echo " ;" > /dev/ttyS1 LED d'état orange clignotante
@ echo "@" > /dev/ttyS1 LED de copie allumée
A echo "A" > /dev/ttyS1 La LED de copie clignote
B echo "B" > /dev/ttyS1 LED de copie éteinte
